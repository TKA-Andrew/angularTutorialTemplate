Introduction

SCSS is a superset of CSS3 syntax, hence, every .css file is a valid .scss file

SCSS is a new syntax for SASS (a pre-processor scripting language).

However, both old(SASS) and new(SCSS) syntaxes have different file extensions and some differences.

Nevertheless, SCSS is prefered in this repo as its syntax is similar to CSS3.

Reference for difference between SCSS and SASS

SCSS with Angular

By default, Angular will use css when new projects are setup.

However, the style can be set when generating new project or component.

For example: ng generate component home --style scss

Use case example

SCSS does help us in improving our code management as it allows us to declare variables.

With the variable declaration function, we can declare global variables for repeatedly used constants.

Example:

            
    $flexContentSize: 0 1 auto;
    $flexRemainingSpace: 1 1 1e-9px;

    .flexContentSize {
    flex: $flexContentSize;
    }

    .flexRemainingSpace {
    flex: $flexRemainingSpace;
    }

    .flexContainer {
    height: 100%;
    display: flex;
    flex-flow: column;
    }
